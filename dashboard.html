livep256-nist
NIST P-256 / secp256r1
The other dominant prime-field curve (TLS, passports). Same EC machinery, no CM/GLV.
11 verified rows · curve + p & n prime
LIVE (second real instance). Kernel-verified: both primes p and n (Pratt certificates, same generator as secp256k1), the unconditional EllipticCurve instance, the base point, and the j≠0/no-GLV structural contrast. Also kernel-verified since: generator order = n (weak keystone), embedding degree (MOV-resistance), non-anomalous trace (SSSA-resistance), and the generic-group Ω(√n) bound (curve-agnostic — the same `generic_dlog_query_bound`, stated for any prime, applies to P-256's n with no new proof). All nine starter-corpus rungs are now verified (domains/p256/corpus.md); the remaining gap is P-256's own strong keystone #E=n — secp256k1's was closed by a curve-specific certificate (CurveCardinalityExact.lean) that exploits j=0 structure P-256 lacks, so P-256 still needs Hasse or a new certificate route. Proves the pipeline is curve-agnostic — Phase-1 exit criterion met on verified facts.
