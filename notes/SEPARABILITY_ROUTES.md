# Closing the `#E[n]=n²` / separability gap — two routes, and the cheap one

The `ψₙ ↔ E[n]` roadmap (`notes/DIVISION_POLY_TORSION_MAP.md`) named **separability of `[n]`** as the
single hard-core barrier. This note decomposes *that* barrier and reaches a decisive, and
**much more optimistic**, conclusion: the payoff facts (`deg[n]=n²`, `#E[n]=n²`) can be reached
by **elementary polynomial algebra over Mathlib's existing `ψₙ`/`Φₙ` API** — *without* the
semester-scale differential/isogeny theory. That makes the gap a **decomposition target**, i.e.
exactly the kind of work the decompose→prove-leaf→autonomy pipeline can close.

## Live Mathlib inventory (checked on the server)
- **Have:** `WeierstrassCurve.ψ, preΨ, ΨSq, Ψ, φ` and the **numerator `Φₙ`** (`Xψₙ² − ψₙ₊₁ψₙ₋₁`),
  plus degrees of `ψ₂/ψ₃/preΨ₄/preΨ'`. General **Kähler differentials `Ω`** (`RingTheory/Kaehler/`).
- **Missing:** the elliptic-curve **invariant differential** as a named object, the **isogeny `[n]`
  as a morphism/field-embedding**, `E[n]≅(ℤ/n)²`, separable-degree = fiber-size.

## The two routes, compared (Fable analysis, cross-checked vs the inventory)

| | **Route A — via the invariant differential** | **Route B — via division polynomials** |
|---|---|---|
| Idea | `[n]*ω = n·ω` ⇒ separable ⇒ `#ker = deg = n²` | `x∘[n] = Φₙ/ψₙ²` in lowest terms ⇒ `deg[n]=n²`; squarefree `ψₙ` ⇒ count |
| Hard cores | **two**: M2 `(φ⊕ψ)*ω=φ*ω+ψ*ω` (large file / formal group) **and** C2 separable-deg=fiber-size (multi-file curve-morphism infra) | **one, and elementary**: B4 squarefree `ψₙ` (a file of recurrence identities, **no new geometry**) |
| New geometric theory needed | invariant differential + finite-morphism fiber theory | none — pure polynomial algebra + one "degree of a rational map" lemma + the `[n]`-as-field-map dictionary (L2, shared, moderate) |
| Cost | semester-scale, human-led | ~2–3 files, each node small enough to **attempt by decomposition** |
| Verdict | long-horizon; reusable payoff (the invariant differential itself) | **recommended** — attempt now |

**Bottleneck if Route A:** M2. **Bottleneck if Route B:** B4 (squarefreeness of `ψₙ`) — "an order of
magnitude cheaper."

| **B4** | `ψₙ` squarefree for `p∤n` (= separability, in polynomial clothing) | MID (file of `gcd(ψₙ,ψₙ')` identities), **no geometry** | the core, but elementary |

Reaching `#E[n]=n²` this way also **discharges the `[IsAddCyclic]` hypothesis** behind the GLV
eigenvalue node (`secp256k1_glvHom_eq_zsmul`) for the prime-order curve — i.e. it closes one of the
explicitly-flagged reductions, upgrading a conditional theorem to an unconditional one.

## Why this matters for the program
- The deep gap is **not** monolithic and **not** blocked on the hardest theory. Route B is
  polynomial algebra — decomposable into leaves that Opus (in-session) and, after decomposition,
  the autonomous harness can close. This is the concrete sense in which **budget + autonomy can
  attack the frontier**: fund the leaf-closing, hand-do the few genuinely subtle nodes (B4, L2).
- Route A remains worthwhile as a **long-horizon, upstream-Mathlib contribution** (the invariant
  differential and separable-degree theory are reusable far beyond this project) — but it is a
  human-led, months-scale effort, honestly flagged.

## Full route analysis (Fable-drafted, pure math)

*Pure algebraic geometry / number theory; none of the nodes below is yet kernel-verified — this is
the plan. Provenance: drafted by Claude Fable 5, cross-checked against the live Mathlib survey.*

**Target (S).** For `E : y²=x³+Ax+B` over `k` (`char k = 0` or `p∤n`): `[n]` separable,
`deg[n]=n²`, `#E[n](k̄)=n²`.

### Route A — via the invariant differential
- **L1 (LEAF, reachable now):** `Ω_{k(E)/k}` is 1-dimensional over `k(E)`, generated by `dx`. Uses
  Kähler differentials of a quotient ring + localization; nonsingularity `4A³+27B²≠0` gives the
  generators proportional and nonzero. Days.
- **L2 (needs one lemma-cluster):** a nonconstant isogeny induces `k(E)↪k(E)` and a pullback `φ*` on
  `Ω`. Functoriality of `Ω` exists; **realizing `[n]` as a `k`-algebra endomorphism of `k(E)` does
  not yet exist** (the curve↔function-field dictionary). Moderate.
- **L3 (reachable modulo L2):** separable ⟺ `φ*` injective ⟺ `φ*ω≠0`. Mathlib has
  formally-unramified ⟺ separable + the fundamental exact sequences.
- **M1 (hard-ish):** `ω=dx/2y ≠ 0`, translation-invariant. A file of `field_simp`/`ring` over the
  addition formulas.
- **M2 (HARD — the heart, true bottleneck):** `(φ⊕ψ)*ω = φ*ω + ψ*ω`. No formal group / scheme-level
  `E`, so a direct two-variable rational-function identity (1–3k lines) or first build the formal
  group of `E` (a multi-file theory). 
- **M3, C1 (trivial given M2,L3):** `[n]*ω = n·ω` by induction; `p∤n ⇒ [n]` separable since `nω≠0`.
- **C2 (HARD — second bottleneck):** separable ⇒ `#ker = deg = n²`. Needs
  separable-degree = generic-fiber-size for finite morphisms of smooth curves — real AG
  infrastructure Mathlib lacks here. Multi-file.
- **M4 (reachable given C2):** `E[n]≅(ℤ/n)²` by finite-abelian-group structure.

**Route A bottom line:** two independent CORE gaps (M2, C2). Semester-scale, human-led.

### Route B — division polynomials, avoiding differentials
- **B1 (MID):** `gcd(Φₙ, ψₙ²)=1` — pure polynomial algebra from the recurrences (Washington-style /
  resultants). A few hundred lines, no geometry.
- **B2 (LEAF-ish, missing name):** `[k(x):k(Φₙ/ψₙ²)] = max(deg Φₙ, deg ψₙ²) = n²` when coprime.
  Standard Gauss-lemma argument; **needs one named lemma**, elementary.
- **B3 (MID):** `deg[n] = [k(E):[n]*k(E)] = n²`, using `[k(E):k(x)]=2` and `[-1]`-symmetry to descend
  to the `x`-line. Needs L2 (to define `deg`), nothing differential.
- **Does B sidestep separability?** For `deg[n]=n²`: **yes, completely.** For `#E[n]=n²`: **no — it
  relocates it** to **B4 (squarefreeness of `ψₙ` for `p∤n`)**, which *is* separability in polynomial
  clothing — but provable elementarily (`gcd(ψₙ,ψₙ')` via recurrence identities), **zero new
  geometric machinery**.

**Route B bottom line:** L2 (shared) + B1 + B2 + B4 — elementary polynomial algebra over the
existing `ψₙ` API, ~2–3 files, each node decomposition-sized. **Recommended.**
