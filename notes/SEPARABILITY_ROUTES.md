# Closing the `#E[n]=n¬≤` / separability gap ‚Äî two routes, and the cheap one

The `œà‚Çô ‚Üî E[n]` roadmap (`notes/DIVISION_POLY_TORSION_MAP.md`) named **separability of `[n]`** as the
single hard-core barrier. This note decomposes *that* barrier and reaches a decisive, and
**much more optimistic**, conclusion: the payoff facts (`deg[n]=n¬≤`, `#E[n]=n¬≤`) can be reached
by **elementary polynomial algebra over Mathlib's existing `œà‚Çô`/`Œ¶‚Çô` API** ‚Äî *without* the
semester-scale differential/isogeny theory. That makes the gap a **decomposition target**, i.e.
exactly the kind of work the decompose‚Üíprove-leaf‚Üíautonomy pipeline can close.

## Why this matters for the program
- The deep gap is **not** monolithic and **not** blocked on the hardest theory. Route B is
  polynomial algebra ‚Äî decomposable into leaves that Opus (in-session) and, after decomposition,
  the autonomous harness can close. This is the concrete sense in which **budget + autonomy can
  attack the frontier**: fund the leaf-closing, hand-do the few genuinely subtle nodes (B4, L2).
- Route A remains worthwhile as a **long-horizon, upstream-Mathlib contribution** (the invariant
  differential and separable-degree theory are reusable far beyond this project) ‚Äî but it is a
  human-led, months-scale effort, honestly flagged.

## Full route analysis (Fable-drafted, pure math)

*Pure algebraic geometry / number theory; none of the nodes below is yet kernel-verified ‚Äî this is
the plan. Provenance: drafted by Claude Fable 5, cross-checked against the live Mathlib survey.*

**Target (S).** For `E : y¬≤=x¬≥+Ax+B` over `k` (`char k = 0` or `p‚à§n`): `[n]` separable,
`deg[n]=n¬≤`, `#E[n](kÃÑ)=n¬≤`.

### Route A ‚Äî via the invariant differential
- **L1 (LEAF, reachable now):** `Œ©_{k(E)/k}` is 1-dimensional over `k(E)`, generated by `dx`. Uses
  K√§hler differentials of a quotient ring + localization; nonsingularity `4A¬≥+27B¬≤‚âÝ0` gives the
  generators proportional and nonzero. Days.
- **L2 (needs one lemma-cluster):** a nonconstant isogeny induces `k(E)‚Ü™k(E)` and a pullback `œÜ*` on
  `Œ©`. Functoriality of `Œ©` exists; **realizing `[n]` as a `k`-algebra endomorphism of `k(E)` does
  not yet exist** (the curve‚Üîfunction-field dictionary). Moderate.
- **L3 (reachable modulo L2):** separable ‚ü∫ `œÜ*` injective ‚ü∫ `œÜ*œâ‚âÝ0`. Mathlib has
  formally-unramified ‚ü∫ separable + the fundamental exact sequences.
- **M1 (hard-ish):** `œâ=dx/2y ‚âÝ 0`, translation-invariant. A file of `field_simp`/`ring` over the
  addition formulas.
- **M2 (HARD ‚Äî the heart, true bottleneck):** `(œÜ‚äïœà)*œâ = œÜ*œâ + œà*œâ`. No formal group / scheme-level
  `E`, so a direct two-variable rational-function identity (1‚Äì3k lines) or first build the formal
  group of `E` (a multi-file theory). 
- **M3, C1 (trivial given M2,L3):** `[n]*œâ = n¬∑œâ` by induction; `p‚à§n ‚áí [n]` separable since `nœâ‚âÝ0`.
- **C2 (HARD ‚Äî second bottleneck):** separable ‚áí `#ker = deg = n¬≤`. Needs
  separable-degree = generic-fiber-size for finite morphisms of smooth curves ‚Äî real AG
  infrastructure Mathlib lacks here. Multi-file.
- **M4 (reachable given C2):** `E[n]‚âÖ(‚Ñ§/n)¬≤` by finite-abelian-group structure.

**Route A bottom line:** two independent CORE gaps (M2, C2). Semester-scale, human-led.

### Route B ‚Äî division polynomials, avoiding differentials
- **B1 (MID):** `gcd(Œ¶‚Çô, œà‚Çô¬≤)=1` ‚Äî pure polynomial algebra from the recurrences (Washington-style /
  resultants). A few hundred lines, no geometry.
- **B2 (LEAF-ish, missing name):** `[k(x):k(Œ¶‚Çô/œà‚Çô¬≤)] = max(deg Œ¶‚Çô, deg œà‚Çô¬≤) = n¬≤` when coprime.
  Standard Gauss-lemma argument; **needs one named lemma**, elementary.
- **B3 (MID):** `deg[n] = [k(E):[n]*k(E)] = n¬≤`, using `[k(E):k(x)]=2` and `[-1]`-symmetry to descend
  to the `x`-line. Needs L2 (to define `deg`), nothing differential.
- **Does B sidestep separability?** For `deg[n]=n¬≤`: **yes, completely.** For `#E[n]=n¬≤`: **no ‚Äî it
  relocates it** to **B4 (squarefreeness of `œà‚Çô` for `p‚à§n`)**, which *is* separability in polynomial
  clothing ‚Äî but provable elementarily (`gcd(œà‚Çô,œà‚Çô')` via recurrence identities), **zero new
  geometric machinery**.

**Route B bottom line:** L2 (shared) + B1 + B2 + B4 ‚Äî elementary polynomial algebra over the
existing `œà‚Çô` API, ~2‚Äì3 files, each node decomposition-sized. **Recommended.**
